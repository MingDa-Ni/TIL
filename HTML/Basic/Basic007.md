### 웹 호스팅

* 웹 서버를 운영할 때
  1. 컴퓨터가 있어야 한다.
  2. 컴퓨터가 항상 켜져 있어야 한다.
  3. 인터넷이 끊겨서는 안된다.
  4. 웹 서버를 설치하고 운영할 수 있어야 한다.
  5. 인터넷을 통해 `외부로 전송`할 수 있게 설정해야 한다.
<br><br>
* 웹 서버를 대신하는 것
   - 인터넷이 연결된 컴퓨터 하나하나를 `호스트(Host)`라고 부른다.
   - 호스트를 빌려주는 사업을 `호스팅(Hosting)`이라고 한다. `클라우드(Cloud)`라고도 부른다.
   - 호스팅 중 웹을 동작시키는 소프트웨어를 빌려주는 비즈니스를 `웹 호스팅(Web Hosting)`이라고 한다.
<br><br>
* 웹 페이지를 보고 싶어하는 방문자들이 웹 페이지를 보는 원리
   1. 깃허브(호스팅 업체)에 파일을 업로드하면 소스코드가 웹 호스팅으로 업로드된다.
   2. 깃허브에서 웹 호스팅 환경에 설치된 `웹 서버를 활성화`한다.
   3. 컴퓨터에 도메인 이름을 부여한다.
   4. 방문자가 도메인 주소로 접속하면 웹 서버가 index.html을 읽는다.
   5. 웹 서버가 방문자에게 `index.html 소스코드를 전송`한다.
<br><br>
* 웹 호스팅 서비스 추천
  - `https://bitballoon.com`
  - `https://neocities.org`
  - Azure Blob
  - Google Cloud Storage
  - Amazon S3

### 웹 서버 
* 웹 브라우저와 웹 서버는 제품군이다.
* 웹 서버에는 `아파치(Apache), IIS, 엔진엑스(Nginx)` 등이 있다.

### 웹 서버 운영
* 파일 열기를 통해 접속했을 때
   - `file://`로 시작된다. 
   - 웹 브라우저가 웹 페이지를 직접 열었다.
   - 웹 서버를 통하지 않는다.
* index.html을 직접 열었을 때
   - `http://`로 시작되는 주소를 입력하고 접속한다.
   - 컴퓨터에 설치된 웹 서버에 접속한다.
   - 웹 서버가 index.html을 읽는다.
   - 웹 브라우저에게 돌려준다.

### http

* http는 `Hyper Text Transfer Protocol`의 약자이다.
  - 웹 페이지를 전송하기 위해 고안된 통신규칙이다.
* 43억개의 IP 주소 중, `127.0.0.1`은 자기 자신을 의미하는 IP 주소이다.
